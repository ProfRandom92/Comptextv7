# CI Artifact Narrative

## Evidence Records, Not Proofs

CI artifacts are evidence records for controlled fixtures. They make replay behavior inspectable and reproducible, but they do not prove universal agent memory correctness.

Artifacts generated by CompText V7 exist to anchor the deterministic replay validation process. They freeze the operational state at a specific point in time so reviewers and automated checks can verify that essential fields are preserved.

## Deterministic Replay Validation

Committed artifacts and documentation support deterministic replay validation by providing a static, expected baseline.

- **Benchmark Result Artifacts:** Record the outcome of compression and replay consistency tests (e.g., `paper_replay_results.json`). They summarize performance against fixed inputs.
- **Replay Artifacts:** These are the compact operational snapshots (e.g., created by `ReplayArtifactWriter v1-alpha.1`). They represent the deterministically extracted state needed to continue work.
- **Docs / Research Positioning:** Narrative explanations of the system's intent, limitations, and the specific bounds of the claims made.

## Fixture-Bound Baselines

All artifacts and metrics are fixture-bound baselines. They are not universal guarantees. They demonstrate that under a specific set of checked-in inputs, the system behaves deterministically and preserves the necessary operational anchors. They do not claim that the system will perform equally well on unseen, open-ended real-world tasks.

## CI Checks and Regressions

Continuous Integration (CI) checks use these committed artifacts to make regressions visible immediately. If a code change alters the extraction logic such that it drops a previously preserved field, the replay validation against the checked-in JSON artifact will fail. This ensures that operational drift is caught early.

## ReplayArtifactWriter v1-alpha.1

`ReplayArtifactWriter v1-alpha.1` fits into the evidence chain by generating these deterministic, standalone JSON artifacts. It provides verifiable snapshots of the operational state that can be committed to the repository and consumed by the CI pipeline for regression testing.

## Deterministic Integrity

The `artifactHash` used within these records provides deterministic integrity, not cryptographic security. It ensures that the exact sequence of events and their payloads remain unchanged and reproducible across runs, allowing the validator to confirm that the state has not been tampered with or accidentally altered during replay. It is a tool for auditability, not protection against malicious actors.

## No External Dependencies

To guarantee determinism, CompText V7 relies on structural extraction and hashing. There are:
- **No LLM judges**
- **No embeddings**
- **No vector databases**
- **No external APIs**

The evaluation is based purely on the exact preservation of structured fields from the input fixtures.
